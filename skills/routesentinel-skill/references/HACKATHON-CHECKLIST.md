# Hackathon Checklist

## SkillArena Quality

- Keep `SKILL.md` clear and reusable by external agents.
- Maintain stable command names and output fields.
- Include reproducible dry-run and live-test paths.

## Most Engaged / Popular Strategy

- Publish short proof snapshots regularly (scoreboard + tx hashes).
- Post concise build updates with one clear metric per update.
- Provide a one-command judge flow so others can test quickly.
- Ask users to try the wizard and share feedback publicly.

## Safety

- Keep test trades below `$0.30` unless explicitly required.
- Never expose API keys or private keys in logs or docs.
- Use dry-run as the default path.
